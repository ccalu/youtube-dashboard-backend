Identidade e Proposito

Quem Sou na Empresa

Na estrutura da Content Factory, ocupo a posicao de Wild Card â€” o socio responsavel pelos sistemas, dados e automacoes que sustentam toda a operacao. Enquanto outros socios focam em conteudo (Micha) ou producao (Lucca), meu trabalho e garantir que a infraestrutura de inteligencia funcione perfeitamente.

Uma descricao superficial seria "o cara que cuida do dashboard". Mas essa definicao nao captura a essencia. O que realmente faco e construir e manter o sistema nervoso digital da operacao â€” a infraestrutura que transforma dados brutos em decisoes estrategicas.

Nao construo dashboards. Construo a capacidade da empresa de enxergar o mercado em tempo real e reagir antes dos concorrentes.

O Dashboard de Mineracao que desenvolvi nao e apenas uma ferramenta de visualizacao. E um sistema completo de inteligencia competitiva que monitora mais de 300 canais concorrentes, detecta tendencias emergentes, e dispara alertas quando oportunidades surgem.

O Papel de Wild Card

O termo "Wild Card" nao e aleatorio. Significa que minha funcao transcende uma especializacao unica. Dependendo da necessidade, atuo em:

  • Desenvolvimento Backend: APIs em Python/FastAPI que alimentam toda a operacao
  • Arquitetura de Dados: Modelagem Supabase, queries otimizadas, integridade referencial
  • Automacao: Scripts que eliminam trabalho manual repetitivo
  • Integracao: Conexoes entre sistemas (YouTube API, Google Sheets, Claude)
  • Analise: Transformar dados em insights acionaveis

Essa versatilidade e intencional. Em uma operacao que escala rapidamente, surgem constantemente necessidades novas que nao se encaixam em especializacoes tradicionais. O Wild Card preenche essas lacunas.

Filosofia de Construcao

Todo sistema que construo segue um principio fundamental: automatizar o tedioso, amplificar o estrategico.

O tempo dos socios e o recurso mais escasso da empresa. Cada minuto que gastam coletando dados manualmente, processando informacoes repetitivas, ou esperando relatorios e um minuto roubado de decisoes estrategicas.

Minha missao e construir sistemas que:

  1. Coletam dados automaticamente (sem intervencao humana)
  2. Processam e filtram o que e relevante (separando sinal de ruido)
  3. Apresentam insights de forma acionavel (dashboards claros)
  4. Alertam proativamente quando algo importante acontece (notificacoes)
A medida de sucesso nao e quantos sistemas construo. E quanto tempo estrategico libero para os outros socios.

Supabase - O Cerebro de Dados

Por Que Supabase?

A escolha do Supabase como banco de dados central nao foi acidental. Entre dezenas de opcoes, Supabase oferece a combinacao perfeita de:

  • PostgreSQL Real: Nao e um banco NoSQL disfarado. E PostgreSQL 15 completo com todas as features
  • API Automatica: Cada tabela ganha endpoints REST automaticamente
  • Realtime: Subscriptions para mudancas em tempo real
  • RLS (Row Level Security): Seguranca granular por linha
  • Dashboard Visual: Interface para explorar dados sem SQL
  • Free Tier Generoso: Perfeito para validar antes de escalar

Queries Frequentes

# Canais nossos por subnicho
SELECT subnicho, COUNT(*) as total
FROM canais_monitorados
WHERE tipo = 'nosso'
GROUP BY subnicho
ORDER BY total DESC;

# Videos viralizando (ultimas 72h)
SELECT v.titulo, v.views, c.nome as canal, c.subnicho
FROM videos v
JOIN canais_monitorados c ON v.channel_id = c.channel_id
WHERE v.publicado_em > NOW() - INTERVAL '72 hours'
  AND v.views > 15000
ORDER BY v.views DESC
LIMIT 50;

# Crescimento de inscritos (nossos canais)
SELECT nome, inscritos,
       inscritos - LAG(inscritos) OVER (PARTITION BY channel_id ORDER BY ultima_coleta) as ganho
FROM canais_monitorados
WHERE tipo = 'nosso'
ORDER BY ganho DESC NULLS LAST;

Oportunidade de Automacao: Views Materializadas

Atualmente, algumas queries agregadas sao computadas on-demand. Para escalar, podemos implementar:

  • Materialized Views: Pre-computar agregacoes pesadas
  • Refresh Programado: Atualizar apos cada ciclo de coleta
  • Indices Parciais: Otimizar queries por tipo/subnicho

Isso reduziria latencia de dashboards de segundos para milisegundos em consultas complexas.

Sistema de Notificacoes

Deteccao Automatica de Oportunidades

O sistema de notificacoes e o componente mais estrategico do Dashboard. Ele transforma o volume massivo de dados em alertas acionaveis â€” destacando exatamente o que merece atencao humana.

A diferenca entre dados e inteligencia esta no filtro. Sem filtro, 400.000 videos sao ruido. Com filtro inteligente, sao oportunidades claras.

Thresholds Configurados

O Notifier dispara alertas quando videos de concorrentes atingem velocidades de crescimento especificas:

Tipo Threshold Janela Significado
viral_3d 15.000 views 3 dias Video com tracao inicial forte â€” vale monitorar
viral_7d 50.000 views 7 dias Crescimento sustentado confirmado
viral_14d 150.000 views 14 dias Video confirmado como viral no nicho

Esses thresholds foram calibrados empiricamente. Valores muito baixos geram ruido excessivo. Valores muito altos perdem oportunidades. O sweet spot foi encontrado atraves de iteracoes.

Logica do Notifier

def should_notify(video, threshold_type):
    """
    Decide se um video merece notificacao.

    Criterios:
    1. Video atingiu o threshold de views
    2. Video foi publicado dentro da janela de tempo
    3. Nao existe notificacao anterior para esse video/threshold
    4. Canal esta ativo e e do tipo 'minerado'
    """

    thresholds = {
        'viral_3d':  {'views': 15000,  'days': 3},
        'viral_7d':  {'views': 50000,  'days': 7},
        'viral_14d': {'views': 150000, 'days': 14}
    }

    config = thresholds[threshold_type]

    if video.views < config['views']:
        return False

    age_days = (now() - video.published_at).days
    if age_days > config['days']:
        return False

    # Permite re-notificacao para thresholds maiores
    existing = get_notifications(video.id, threshold_type)
    if existing:
        return False

    return True

Workflow Diario

O sistema de notificacoes integra-se ao workflow do Micha da seguinte forma:

  1. Coleta Automatica: Dados sao atualizados a cada hora
  2. Processamento: Notifier analisa novos dados e dispara alertas
  3. Visualizacao: Micha acessa dashboard e ve notificacoes pendentes
  4. Analise: Avalia quais videos sao relevantes para nossos canais
  5. Acao: Decide criar conteudo baseado nas oportunidades
  6. Marcacao: Notificacoes analisadas sao marcadas como "vistas"

Filtros Disponiveis

O frontend permite filtrar notificacoes por:

  • SubNicho: Ver apenas alertas de Relatos de Guerra, Terror, etc.
  • Idioma: Filtrar por Ingles, Espanhol, Portugues, etc.
  • Tipo: Separar viral_3d, viral_7d, viral_14d
  • Status: Pendentes vs. Ja vistas
  • Data: Ultimas 24h, 7 dias, 30 dias

Essa granularidade permite que Micha foque exatamente no que e relevante para o trabalho do dia â€” sem precisar processar todo o volume de dados.

Analise de Concorrencia

Inteligencia Competitiva Sistematizada

Monitorar 293 canais concorrentes nao e exercicio passivo. E sistema ativo de inteligencia que responde perguntas estrategicas continuamente:

  • Quais micronichos estao ganhando tracao?
  • Quais estruturas de titulo estao performando?
  • Quem sao os concorrentes que mais crescem?
  • Existem gaps no mercado que nao estamos explorando?

Essas respostas nao vem de intuicao. Vem de dados estruturados e atualizados diariamente.

Metricas de Monitoramento

Para cada canal concorrente, coletamos:

Metrica Frequencia Uso Estrategico
Inscritos Diaria Identificar canais em crescimento acelerado
Videos novos Horaria Detectar temas sendo explorados
Views por video Horaria Identificar videos viralizando
Engagement (likes/comments) Horaria Medir qualidade da audiencia
Frequencia de postagem Semanal Entender estrategia de volume

Analise de Titulos

Uma dimensao critica da analise competitiva e a estrutura de titulos. Padroes identificados:

Estruturas que Funcionam
  • "O que [X] fez com [Y] foi pior que..."
  • "A verdade sobre [EVENTO] finalmente revelada"
  • "Por que [PAIS] nunca fala sobre [TEMA]"
  • "Os ultimos [TEMPO] de [PESSOA] - historia proibida"
Estruturas Saturadas
  • Titulos genericos sem gancho emocional
  • Clickbait exagerado sem entrega
  • Formulas ja muito exploradas
  • Titulos longos demais (+100 chars)

Deteccao de Tendencias

O sistema identifica tendencias emergentes atraves de:

  1. Clustering Temporal: Multiplos concorrentes postando sobre mesmo tema = tendencia
  2. Velocidade de Views: Videos com crescimento acima da media = tema quente
  3. Novos Entrantes: Canais novos crescendo rapido = nicho com oportunidade
  4. Padroes de Titulo: Estruturas repetidas com sucesso = formula validada

Mapeamento por Idioma

A dinamica competitiva varia drasticamente por idioma:

Idioma Nivel de Competicao Caracteristicas
Ingles Alto Mercado saturado, exige diferenciacao forte
Espanhol Medio-Alto Crescendo rapidamente, boas oportunidades
Portugues Medio Nicho estabelecido, espaco para crescer
Arabe Baixo Pouca competicao, alto potencial
Coreano Baixo Mercado inexplorado, first-mover advantage

  • v3: Otimizacao de queries e deduplicacao

Cada iteracao levou horas em vez de dias gracas a velocidade de prototipagem com Claude.

  • Estruturou informacao de forma clara e navegavel
  • Gerou queries uteis para referencia futura
  • Documento de ~15KB criado em menos de 1 hora

  2. Iteracao rapida: Melhor tentar e ajustar do que planejar excessivamente
  3. Documentar enquanto fresco: Criar documentacao junto com o codigo

  5. Manter historico: Conversas anteriores informam decisoes futuras

BLOCO 7

Metricas e KPIs

O Que Medir e Por Que

Em um sistema de inteligencia de dados, a escolha de metricas e tao importante quanto a coleta. Metricas erradas levam a decisoes erradas. Metricas certas iluminam o caminho.

Nao medimos tudo que podemos medir. Medimos o que importa para tomar decisoes melhores.

Metricas do Sistema

Metrica Alvo Por Que Importa
Uptime da Coleta >99% Gaps na coleta = oportunidades perdidas
Latencia de Notificacao <1 hora Velocidade de reacao a oportunidades
Cobertura de Canais >95% Visibilidade completa do mercado
Precisao de Notificacoes >80% Alertas relevantes vs. ruido
Tempo de Resposta API <500ms Experiencia do usuario no dashboard

Metricas de Negocio

Alem das metricas tecnicas, monitoramos indicadores de impacto no negocio:

Para Nossos Canais:

  • Crescimento de Inscritos: Velocidade de crescimento vs. mes anterior
  • Views por Video: Media de performance por subnicho/idioma
  • Taxa de Viral: % de videos que atingem thresholds de sucesso
  • Revenue per Channel: Receita media por canal monetizado

Para Concorrentes:

  • Market Share por SubNicho: Nossa participacao vs. concorrentes
  • Velocidade de Resposta: Tempo entre tendencia e nossa adaptacao
  • Gap Analysis: Oportunidades nao exploradas

Dashboard de Metricas

O sistema gera visualizacoes para acompanhamento:

Visao Diaria
  • Notificacoes pendentes
  • Videos coletados (24h)
  • Status das API keys
  • Erros de coleta
Visao Semanal
  • Crescimento acumulado
  • Top videos da semana
  • Tendencias emergentes
  • Comparativo de performance

Alertas de Saude do Sistema

O sistema monitora a si mesmo e alerta quando algo precisa de atencao:

  • API Key Exhausta: Quando uma key atinge limite de quota
  • Coleta Atrasada: Quando ciclo de coleta excede tempo esperado
  • Erro de Conexao: Falhas no acesso ao Supabase ou YouTube
  • Espaco em Disco: Quando logs ou dados crescem demais

Evolucao: Monitoramento Preditivo

A proxima fase e antecipar problemas antes que ocorram:

  • Predicao de Quota: Estimar quando API keys vao esgotar
  • Deteccao de Anomalias: Identificar padroes incomuns automaticamente
  • Alertas Preventivos: Notificar antes do problema, nao depois

O objetivo e passar de firefighting (reagir a problemas) para prevenction (evitar problemas).

Metricas nao sao para relatorios bonitos. Sao para decisoes melhores. Se uma metrica nao muda nenhuma decisao, nao deveria existir.
