2. Score, Filtros e Dados

Score: 50% CTR + 50% Views

Cada video recebe um score de 0 a 100, normalizado min-max dentro do canal. Metade do peso e o CTR (capacidade de atrair clique) e metade e as views brutas (alcance real).

Video com 145K views e 8.2% CTR = Score 85/100
Video com 12K views e 4.1% CTR = Score 22/100

Filtros

Nem todo video e analisado. Filtros garantem que so videos com dados relevantes entram na analise.

Minimo 7 dias de vida (maturidade do video)
Minimo 500 views (abaixo disso nao vale analisar)
Minimo 5 videos no canal (amostra minima)
CTR relativo: cada video mostra o CTR comparado com a media do canal. Ex: "CTR: 8.2% (canal: 6.4% | +1.8pp)" -- isso mostra se o video performa acima ou abaixo do canal.

3. Arquitetura do Agente

PYTHON (Dados + Numeros)

  • Busca todos os videos do canal
  • Aplica filtros (7 dias, 500 views)
  • Busca CTR de cada video + CTR medio do canal (referencia, nao entra no score)
  • Calcula score (50% CTR + 50% Views)
  • Organiza ranking por score
  • Detecta videos novos vs ja analisados
  • Calcula deltas (Views +X%, CTR +Xpp)
  • Conta motores e suas estatisticas
  • Salva tudo no banco de dados

LLM (Analise + Inteligencia)

  • LLM TEMAS: Extrai tema concreto de cada video
  • LLM TEMAS: Identifica hipoteses de motores
  • LLM MOTORES: Comenta cada video do ranking
  • LLM MOTORES: Identifica motores dominantes
  • LLM MOTORES: Descobre padroes e tendencias
  • LLM MOTORES: Gera recomendacoes acionaveis
  • LLM MOTORES: Compara com relatorios anteriores
Regra de ouro: Python faz 100% dos calculos numericos (scores, rankings, deltas, contagens). A LLM NUNCA inventa ou altera numeros -- ela recebe dados prontos e faz a analise qualitativa.

Sempre 1 relatorio unico por execucao. A analise do dia vem PRIMEIRO (novos videos, motores, padroes, recomendacoes). A comparacao com anteriores vem DEPOIS, como secao separada no final.

8. Gestao de Relatorios

#

Numeracao Sequencial

Cada relatorio recebe um numero sequencial por canal: #1, #2, #3... com a data de execucao. Facil de identificar e referenciar.

X

Excluir Relatorio

"Excluir relatorio #2" -- remove o relatorio e renumera os seguintes. Videos daquele relatorio voltam a ser "novos" na proxima analise.

R

Refazer Analise

Excluir o ultimo relatorio e rodar novamente. Util para testar ajustes nos prompts ou quando a LLM nao entregou qualidade suficiente.

Exemplo pratico: Canal tem relatorios #1 (10/mar), #2 (17/mar), #3 (24/mar). Voce exclui o #2. Resultado: #1 (10/mar) e #2 (24/mar, renumerado). Os videos que estavam no antigo #2 serao re-analisados na proxima execucao.

9. Deteccao Incremental -- Como Evita Duplicacao

Cada relatorio salva um snapshot (analyzed_video_data) com os dados de cada video no momento da analise. Na proxima execucao, o Python compara os dados atuais com o snapshot:

Situacao Como detecta O que acontece
NOVO video_id NAO existe no snapshot Vai pra LLM TEMAS --> extrai tema + hipoteses
ATUALIZADO Views +20% OU CTR +2pp vs snapshot Python atualiza numeros. LLM MOTORES recebe os deltas na comparacao
SEM MUDANCA Abaixo dos thresholds Ignorado. Mantem tema/hipoteses/score do snapshot

Resultado: Zero Duplicacao

A LLM TEMAS so recebe videos NOVOS. Nunca re-analisa o mesmo video. Os numeros (views, CTR, score) sao atualizados 100% pelo Python -- precisos e sem custo de LLM. A LLM MOTORES recebe tudo pronto para gerar a analise narrativa.
