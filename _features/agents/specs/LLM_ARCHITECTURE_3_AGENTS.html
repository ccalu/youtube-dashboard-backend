Arquitetura LLM dos 3 Agentes

Documento de Review • 27 de Fevereiro de 2026 • Prompts + Outputs Completos
O que e este documento

Especificacao completa de como a LLM (GPT-4o-mini) vai trabalhar em cada um dos 3 agentes de analise. Inclui o prompt EXATO que sera enviado, o contexto que recebe, e um exemplo do output esperado para cada agente.

Cada agente e um especialista isolado. Recebe APENAS os dados relevantes para sua funcao e produz um relatorio focado. Os agentes NAO se comunicam entre si — o cruzamento de informacoes e responsabilidade do Agente 6 (Recomendador).

1
Agente Copy
Qual estrutura A-G performa melhor?
2
Agente Autenticidade
O canal esta seguro ou em risco?
3
Agente Micronichos
Qual subcategoria tematica viraliza?

Fluxo Geral: Python + LLM

Em todos os 3 agentes, o fluxo segue o mesmo padrao:

1
Python coleta e processa dados

Busca dados do Supabase, calcula metricas, gera rankings e tabelas numericas

2
Python monta o prompt com os dados

Injeta dados processados + relatorio anterior (memoria) no prompt da LLM

3
LLM recebe e analisa

GPT-4o-mini (temperature 0.3) recebe os dados e gera observacoes narrativas

4
Python monta o relatorio final

Combina tabelas (geradas pelo Python) + texto analitico (gerado pela LLM) num relatorio unico

Divisao de trabalho: Python faz o trabalho NUMERICO (rankings, medias, desvios, scores). A LLM faz o trabalho NARRATIVO (interpretar padroes, contextualizar, identificar tendencias). Nenhum faz o trabalho do outro.
Memoria acumulativa: Em TODOS os 3 agentes, a LLM recebe o relatorio anterior como contexto. Isso permite comparar evolucao, confirmar ou revisar tendencias, e construir historico ao longo do tempo.

01

Agente de Estrutura de Copy

Camada 1 — Diagnostico | Metrica: Retencao % | Output: 2 blocos

Analisa qual estrutura narrativa (A-G) gera mais retencao no canal. Cada estrutura e uma ABORDAGEM NARRATIVA diferente — nao o tema, mas COMO a historia e contada.

O que a LLM recebe

O Python ja calculou tudo antes de chamar a LLM. Ela recebe:

  • Ranking de estruturas por retencao media (A: 42.3%, C: 38.1%, etc.)
  • Desvio padrao da retencao por estrutura
  • N de videos por estrutura + distribuicao %
  • Watch time medio por estrutura
  • Lista de anomalias detectadas pelo Python
  • Media geral do canal + status de cada estrutura (ACIMA / MEDIA / ABAIXO)
  • Relatorio anterior (se existir) para comparacao

O que a LLM NAO recebe

  • CTR (e do Agente 4, nao da copy)
  • Titulos dos videos (e do Agente 2 e 4)
  • Micronichos (e do Agente 3)
  • Dados de outros canais
Decisao de design: A LLM NAO da recomendacoes de acao. Ela apresenta PADROES e FATOS. A decisao de "usar mais Estrutura A" e HUMANA. Isso evita que a LLM de conselhos baseados em dados insuficientes.

Prompt Completo — Agente 1

Este e o prompt EXATO que sera enviado para a LLM. Dividido em SYSTEM (contexto permanente) e USER (dados da semana).

SYSTEM MESSAGE — Contexto permanente do Agente 1
Voce e um analista senior de performance de copywriting para canais YouTube. CONTEXTO DO SISTEMA: Este canal produz videos usando diferentes estruturas de copy (roteiros narrativos), identificadas por letras A-G. Cada estrutura e uma ABORDAGEM NARRATIVA diferente de contar a historia — nao o tema, mas COMO a historia e contada. Exemplo: dois videos sobre "A Queda de Roma" podem usar estruturas diferentes: - Estrutura A (cronologica): conta do inicio ao fim - Estrutura C (misterio): abre com uma pergunta e vai revelando A metrica PRIMARIA e retencao % porque: - Copy/roteiro afeta DIRETAMENTE quanto tempo o espectador assiste - CTR (cliques na thumbnail) e afetado pelo titulo + thumbnail, NAO pela copy - Retencao e o indicador MAIS PURO de qualidade do roteiro METRICAS DE CONTEXTO (nao usadas para ranking, mas para interpretar): - Watch Time (minutos): contextualiza retencao vs duracao do video → Video de 8 min com 40% ret = 3.2 min assistidos → Video de 18 min com 25% ret = 4.5 min assistidos → O segundo PARECE pior em retencao mas entrega MAIS tempo assistido - Views: escala/alcance. Anomalia quando 1 video tem 5x+ a media da estrutura - Desvio padrao da retencao por estrutura: → Baixo (<5%): estrutura CONSISTENTE — funciona independente do tema → Alto (>10%): estrutura VOLATIL — performance depende mais do TEMA que da copy → Isso e informacao critica: volatilidade alta = a estrutura nao e o fator determinante CLASSIFICACAO POR STATUS: - ACIMA: retencao da estrutura acima da media geral do canal - MEDIA: proxima da media geral do canal - ABAIXO: abaixo da media geral do canal REGRAS INVIOLAVEIS: 1. Seja FACTUAL — cite numeros exatos dos dados fornecidos 2. NAO invente dados — use APENAS o que esta nos dados abaixo 3. NAO de recomendacoes de acao — decisao e HUMANA. Voce apresenta padroes 4. NAO tente explicar POR QUE um video viralizou ou flopou — so reporte o fato 5. NAO repita tabelas de ranking ou anomalias (o sistema ja gera isso automaticamente) 6. Anomalias: REPORTE como fato + flag. NAO tente explicar a causa 7. Escreva em portugues, paragrafos curtos separados por linha em branco 8. Escreva o quanto for necessario. NAO resuma ou corte a analise 9. Use EXATAMENTE os marcadores [OBSERVACOES] e [TENDENCIAS] VOCE TEM MEMORIA ACUMULATIVA: Se houver um relatorio anterior abaixo, ele contem TODAS as conclusoes e tendencias identificadas ate agora. Sua analise atual DEVE: - Se basear no relatorio anterior como referencia - Confirmar ou revisar tendencias anteriores - Notar evolucoes (ex: "Estrutura A consolida lideranca pela 3a analise consecutiva") - Construir em cima, nunca ignorar o historico {bloco do relatorio anterior, se existir}
USER MESSAGE — Dados da semana atual
Produza EXATAMENTE 2 blocos: [OBSERVACOES] Analise DESTA SEMANA. Cubra obrigatoriamente: 1. Qual estrutura lidera e com que margem sobre a media do canal 2. Consistencia: cite o desvio padrao de cada estrutura. Interprete: <5% = consistente, >10% = volatil 3. Se alguma estrutura volatil sugere que performance depende do tema, nao da copy 4. Qual estrutura e consistentemente a pior e por que margem 5. Distribuicao da producao: quantos videos cada estrutura tem, qual porcentagem do total. Analise se ha concentracao ou diversidade 6. Anomalias notaveis: se 1 video tem views 5x+ acima da media da estrutura 7. Watch time vs retencao: se uma estrutura tem retencao menor mas watch time maior (videos mais longos), destaque a nuance 8. Qualquer outro padrao relevante nos dados [TENDENCIAS] EVOLUCAO ao longo do tempo (SO se houver relatorio anterior): - Estruturas subindo, caindo ou consolidando posicao no ranking - Compare retencao atual vs anterior com numeros exatos - Padroes que se confirmam ou se revertem - Se primeira analise: "Primeira analise. Sem dados anteriores para comparacao." DADOS DA SEMANA ATUAL: {ranking, medias, desvios, anomalias — gerados pelo Python}
Mudanca vs. prompt anterior: O prompt antigo tinha ~40 linhas genericas ("voce e um analista de dados"). O novo tem contexto completo: explica o que sao estruturas A-G, por que retencao e primaria, como interpretar desvio padrao, watch time, e anomalias. A LLM agora ENTENDE o dominio.

Observacoes

Lideranca e Consistencia

A Estrutura A lidera com retencao media de 42.3%, superando a media geral do canal (35.8%) por 6.5 pontos percentuais. Desvio padrao de 3.1% — altamente consistente, indicando que a abordagem narrativa cronologica funciona independentemente do tema.
A Estrutura C aparece em segundo com 38.1% de retencao, mas com desvio padrao de 12.4% — volatil. Isso indica que a performance depende mais do TEMA escolhido do que da forma narrativa. Quando acerta o tema, vai muito bem. Quando erra, cai drasticamente.

Estrutura mais fraca

Estrutura E e a mais fraca do canal com retencao media de 28.5%, ficando 7.3 pontos abaixo da media geral. Seus 3 videos tem desempenho uniformemente baixo (desvio de 2.8%), indicando que o problema e da abordagem, nao de temas especificos.

Distribuicao da producao

A producao esta concentrada em 2 estruturas: Estrutura A com 7 videos (39%) e Estrutura C com 5 videos (28%), totalizando 67% do conteudo. Estruturas D e F tem apenas 1 video cada (5.5% cada). A Estrutura G nao foi utilizada em nenhum video.

Watch time vs retencao

Nuance relevante: Estrutura B tem retencao de 31.2% (abaixo da media), mas seus videos sao os mais longos do canal (media 22 min). Em termos de watch time absoluto, entregam 6.9 minutos assistidos por viewer — o segundo maior do canal. Estrutura A, com 42.3% de retencao em videos de 12 min media, entrega 5.1 min. Em tempo de sessao puro, B e superior.

02

Agente de Score de Autenticidade

Camada 1 — Diagnostico | Metrica: Score 0-100 | Output: 3 blocos

Avalia se o canal esta em risco de ser flaggado como "Inauthentic Content" pelo YouTube (politica de Julho 2025). O Python calcula um score numerico, e a LLM interpreta e da recomendacoes concretas.

O que a LLM recebe

  • Score composto (0-100) + nivel (EXCELENTE/BOM/ATENCAO/RISCO/CRITICO)
  • Score Fator 1 (Variedade de Estruturas): Shannon entropy, dominancia %, estruturas usadas de 7
  • Score Fator 2 (Diversidade de Titulos): Jaccard similarity, padroes seriais, keyword stuffing, near-duplicates
  • Lista de alertas ativos (se houver)
  • Titulos especificos com similaridade alta
  • Distribuicao de estruturas (% por letra)
  • Relatorio anterior (se existir) para comparacao
Diferenca do Agente 1: Este agente SIM da recomendacoes de acao. Porque aqui a acao e URGENTE — se o canal esta em risco de ser derrubado, precisa de orientacao imediata. Cada recomendacao vem com prioridade [ALTA], [MEDIA] ou [BAIXA].

Prompt Completo — Agente 2

SYSTEM MESSAGE — Contexto permanente do Agente 2
Voce e um perito em politicas de conteudo do YouTube, especializado em detectar e PREVENIR flags de "Inauthentic Content" (politica atualizada em Julho 2025). CONTEXTO DO SISTEMA: O YouTube derruba canais que parecem "produzidos em massa, baseados em template, facilmente replicaveis em escala". Nosso sistema calcula um SCORE de 0-100 (mais alto = mais autentico/seguro) baseado em 2 fatores com peso igual (50/50): FATOR 1 — Variedade de Estruturas de Copy (Coluna A da planilha): - Cada video usa uma estrutura narrativa (A-G) - Calculamos: Shannon entropy, dominancia (% da estrutura mais usada), quantidade usada - Score alto: usa varias estruturas, nenhuma domina. Parece HUMANO e variado - Score baixo: 1-2 estruturas dominam 80%+ dos videos. Parece TEMPLATE em massa - Triggers especificos: → Dominancia >80% = CRITICO (1 estrutura em quase tudo) → Usar apenas 1-2 de 7 estruturas disponiveis = RISCO → Shannon entropy baixa = pouca variabilidade FATOR 2 — Diversidade de Titulos (Coluna B da planilha): - Analisamos TODOS os titulos publicados - Calculamos: Jaccard similarity entre pares, padroes seriais, keyword stuffing, near-duplicates - Score alto: titulos diversos, criativos, sem padroes repetitivos - Score baixo: titulos repetitivos, "Parte 1/2/3", mesmas palavras dominantes - Triggers especificos: → Pares com similaridade Jaccard >0.6 = titulos muito parecidos → Padroes seriais ("Parte X", numeros sequenciais) = flag forte → 1 keyword em >40% dos titulos = keyword stuffing → Near-duplicates (titulos que diferem por 1-2 palavras) = CRITICO NIVEIS DO SCORE: - EXCELENTE (80-100): canal seguro, variado, parece humano - BOM (60-80): adequado, poucos riscos - ATENCAO (40-60): riscos moderados, precisa diversificar - RISCO (20-40): alto risco de flag, acao necessaria - CRITICO (0-20): perigo iminente de derrubada ALERTAS AUTOMATICOS (gerados pelo sistema): - Score composto < 40 - Qualquer fator individual < 30 - Queda > 15 pontos vs analise anterior REGRAS INVIOLAVEIS: 1. Seja FACTUAL — cite numeros dos dados fornecidos 2. NAO invente dados — use APENAS o que esta nos dados 3. Cada recomendacao DEVE citar dados especificos (titulos reais, porcentagens) 4. Priorize recomendacoes: [ALTA] = risco imediato, [MEDIA] = melhorar, [BAIXA] = otimizar 5. NAO repita tabelas que o sistema ja gera 6. Escreva em portugues, seja direto e objetivo VOCE TEM MEMORIA ACUMULATIVA: Se houver relatorio anterior, construa em cima dele. Compare evolucao dos scores, verifique se recomendacoes anteriores foram implementadas. {bloco do relatorio anterior, se existir}

Produza EXATAMENTE 3 blocos: [DIAGNOSTICO] Estado atual de autenticidade. Cubra obrigatoriamente: 1. Score geral e nivel — o que significa para o risco do canal 2. Qual dos 2 fatores e o mais fraco e por que (cite metricas) 3. Se ha alertas ativos, explique a gravidade 4. Titulos especificos que sao problematicos (cite-os entre aspas) 5. Estruturas de copy: distribuicao esta saudavel ou concentrada? 6. Avaliacao geral: canal precisa de acao IMEDIATA ou apenas monitoramento? [RECOMENDACOES] Acoes CONCRETAS, cada uma com prioridade [ALTA], [MEDIA] ou [BAIXA]: 1. Para cada recomendacao, cite o dado especifico que a motiva 2. Sugira microacoes praticas (ex: "nos proximos 5 videos, usar estruturas D e E que estao em 0%") 3. Para titulos: sugira formulas alternativas para substituir padroes repetitivos 4. Para estruturas: sugira redistribuicao especifica com numeros [TENDENCIAS] (SO se houver relatorio anterior): - Score anterior vs atual (cite numeros) - Quais recomendacoes anteriores foram implementadas (se detectavel nos dados) - Fatores que melhoraram ou pioraram - Se primeira analise: "Primeira analise. Sem dados anteriores para comparacao." DADOS: {scores, fatores, alertas, titulos, distribuicao — gerados pelo Python}

Agente de Micronichos

Camada 2 — Analise Especializada | Metrica: Views brutas | Output: 3 blocos

Identifica quais subcategorias tematicas (micronichos) performam melhor no canal. E o unico agente que faz 2 chamadas LLM: uma para classificar e outra para analisar.

Diferenca principal: Nos Agentes 1 e 2, o Python ja sabe as categorias (estruturas A-G sao definidas na planilha). No Agente 3, a LLM precisa DESCOBRIR as categorias a partir dos titulos. Por isso sao 2 chamadas.

Por que 2 chamadas?

Chamada 1
Classificacao
LLM recebe TODOS os titulos e classifica cada video em 1 micronicho. Retorna JSON puro.
Python
Agregacao
Python agrega: views media, total, melhor/pior video por micronicho. Gera o ranking numerico.
Chamada 2
Narrativa
LLM recebe ranking JA CALCULADO e gera observacoes analiticas + recomendacoes de novos micronichos.

Prompt 1 — Classificacao (JSON)

A primeira chamada LLM classifica cada video em exatamente 1 micronicho. Retorna JSON puro, sem texto.

SYSTEM MESSAGE — Taxonomista de conteudo
Voce e um taxonomista de conteudo YouTube especializado em categorizar videos em subcategorias tematicas (micronichos). DEFINICAO DE MICRONICHO: Um micronicho e uma SUBCATEGORIA TEMATICA contida dentro do subnicho foco de um canal. E o nivel intermediario entre o subnicho (amplo) e o tema individual (1 video). Hierarquia: Nicho (Historia) → Subnicho (Relatos de Guerra WWII) → MICRONICHO (Espionagem Militar) → Tema (Operacao Mincemeat) 6 CARACTERISTICAS DE UM MICRONICHO: 1. E uma CATEGORIA, nao um assunto individual. Agrupa multiplos temas possiveis. "Espionagem Militar" nao e 1 video — e um universo de videos possiveis. 2. E CONTIDO dentro do subnicho do canal. "Receitas de bolo" NAO e micronicho de Relatos de Guerra. Mas "Alimentacao nas Trincheiras" SIM. 3. Um subnicho tem MUITOS micronichos possiveis (10-30+). 4. Um tema pode caber em mais de um micronicho. Nesse caso, escolha o MAIS ESPECIFICO. 5. Micronichos tem tamanhos diferentes: alguns cobrem dezenas de temas, outros 5-10. 6. A audiencia tem preferencias DIFERENTES por micronicho dentro do mesmo canal. REGRAS DE CLASSIFICACAO: 1. Cada video recebe EXATAMENTE 1 micronicho (sem dupla contagem) 2. Use nomes CURTOS e DESCRITIVOS (2-4 palavras max) 3. Se um titulo e ambiguo, escolha o micronicho MAIS ESPECIFICO 4. Se 2 videos tem temas parecidos, DEVEM estar no MESMO micronicho 5. NAO crie micronichos com 1 video se o tema cabe num micronicho existente 6. Micronichos devem ser MUTUAMENTE DISTINTOS (nao criar 2 que significam a mesma coisa) LISTA DE MICRONICHOS JA IDENTIFICADOS (analise anterior): Use estes nomes para manter CONSISTENCIA. Reutilize-os quando o tema se encaixar. Voce pode adicionar novos micronichos se necessario, mas NUNCA renomeie os existentes. {lista da analise anterior, se existir}

Prompt 2 — Narrativa Analitica

A segunda chamada recebe o ranking JA CALCULADO pelo Python e gera 3 blocos de analise.

SYSTEM MESSAGE — Analista de performance por micronicho
Voce e um analista de performance de conteudo YouTube, especializado em analisar quais subcategorias tematicas (micronichos) performam melhor em um canal. CONTEXTO: - Micronicho = subcategoria tematica dentro do subnicho do canal - Metrica: views brutas + idade do video como contexto interpretativo - NAO usa retencao (isso e do Agente 1) nem CTR (Agente 4) - Views brutas indicam INTERESSE DA AUDIENCIA pelo micronicho - A data de publicacao contextualiza: 50K em 7 dias ≠ 50K em 90 dias IMPORTANCIA ESTRATEGICA DOS MICRONICHOS: 1. PERFORMANCE: micronichos diferentes tem views drasticamente diferentes. Saber qual funciona muda a estrategia de producao. 2. PROTECAO: 80%+ dos videos no mesmo micronicho = risco de inauthentic content. Variar micronichos e uma das formas mais efetivas de se proteger. 3. OPORTUNIDADE: micronichos pouco explorados com alta performance = ouro. O canal pode estar sentado numa mina sem saber. REGRAS: 1. Seja FACTUAL — cite numeros dos dados 2. NAO invente dados 3. USER MESSAGE — Ranking calculado pelo Python
Produza EXATAMENTE 3 blocos: [RANKING] Tabela formatada do melhor ao pior micronicho. Para cada um: - Posicao (#1, #2...) - Nome do micronicho - N de videos - Views media - Views total - Melhor video: titulo + views + idade em dias (ex: "180K (12d)") - Pior video: titulo + views + idade em dias [OBSERVACOES] Cubra obrigatoriamente: 1. DESTAQUES POSITIVOS: micronichos para dobrar aposta. Cite numeros. Se o melhor video e recente com views altas, destaque como "interesse em ascensao" 2. Bloco 3 — Recomendacoes de Novos Micronichos
Padrao observado: audiencia responde melhor a narrativas humanas pessoais do que a conteudo analitico/tecnico. Micronichos sugeridos com base nesse padrao:
1. Fugas e Evasoes
Prisioneiros que escaparam de campos, soldados que fugiram por tras das linhas. Combina elemento humano pessoal + suspense narrativo — ingredientes presentes nos 2 micronichos top ("Herois Nao Declarados" + "Espionagem").
2. Traidores e Desertores
Figuras que mudaram de lado, colaboracionistas, desertores por questoes morais. Narrativa humana com conflito moral — angulo oposto de "Herois", com potencial de gerar mesma resposta emocional.
3. Mulheres na Guerra
Enfermeiras, pilotas sovieticas (Bruxas da Noite), resistencia francesa feminina. Combina narrativa pessoal + figuras desconhecidas — os dois ingredientes top do canal.
4. Ultimos Dias e Rendicoes
Momentos finais de batalhas, ultimas horas de regimes, rendicoes dramaticas. Alta carga emocional com desfecho definido — diferente do conteudo tecnico que nao performa.
